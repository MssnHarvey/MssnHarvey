DC-1靶机渗透测试报告

发表于2021-03-03|更新于2021-03-17|渗透
|阅读量:

立个flag：一天一个DC机（滑稽

准备工作

DC-1（nat模式）；kali（nat模式）

信息搜集

打开DC-1页面如下：（没给账号密码需要自己渗透进去

首先我们对其进行内网扫描


这里我直接浏览器对其访问发现的

Wappalyzer查看其服务信息发现cms是Drupal 7

或者用nmap扫描

接着扫波目录，发现都没啥卵用

漏洞攻击

msf搜索目标cms版本漏洞，实现渗透


emmm，这里我们就用最新的漏洞

芜湖，exit后选择4（第二新的）成功getshell

反弹式的shell，进入交互界面

cat查看flag1.txt得到flag1

Every good CMS needs a config file =》 每个好的CMS都需要一个配置文件
于是乎我们去查查他的配置文件得到flag2

这里我们成功拿到数据库的账号密码，所以我们就登录进数据库

接着查询数据库信息

接着查表

发现关键信息users，查询里面的字段

得出账号密码，但是密码是hash加密过的，所以我们只能生成一个密码并得到其hash值进行替换

最后成功登录进去得到flag3

提示我们：特殊的PERMS可以帮助找到密码-但您需要-exec该命令来确定如何获取隐藏的内容。
通过 find / -perm -4000 寻找权限是-4000的文件即查找有特殊权限的命令

利用find提权获取root权限，得到flag4
